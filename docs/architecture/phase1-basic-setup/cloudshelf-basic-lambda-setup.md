## üìö Best Practices & Troubleshooting

<details>
<summary><strong>‚ö° Phase 1 Lambda Best Practices</strong></summary>

### **Function Design**

- ‚úÖ **Keep functions simple** - Single responsibility per function
- ‚úÖ **Use environment variables** - For table names and configuration
- ‚úÖ **Handle errors gracefully** - Return meaningful error messages
- ‚úÖ **Log important events** - Use context.getLogger() for CloudWatch logging

### **DynamoDB Integration**

- ‚úÖ **Use AWS SDK v2** - Modern Java SDK with better performance
- ‚úÖ **Handle AttributeValue types** - Proper string/number conversions
- ‚úÖ **Plan for scaling** - Design partition keys for even distribution
- ‚úÖ **Monitor costs** - Watch read/write capacity consumption

### **Common Issues**

**Function Permission Errors**:

```
ERRORLOG: User is not authorized to perform: dynamodb:PutItem
```

- **Solution**: Verify IAM role has `AmazonDynamoDBFullAccess` attached

**Table Not Found Errors**:

```
ERRORLOG: Requested resource not found
```

- **Solution**: Check table name matches exactly: `CloudShelf-ShoppingCart`

**JSON Parsing Errors**:

```
ERRORLOG: No JSON object could be decoded
```

- **Solution**: Ensure API Gateway sends proper JSON in event body

</details>
